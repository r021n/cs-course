Menyimpan dan Mengorganisir Data yang Kompleks

Setelah berhasil mengorganisir logika programnya dengan fungsi, Brando menghadapi tantangan baru. Ibunya sekarang menjual lebih dari sepuluh jenis kue, dan setiap kue memiliki nama, harga, dan stok yang berbeda. Menyimpan semua data ini dalam variabel-variabel tunggal akan menjadi kacau. Brando tahu ia membutuhkan struktur data yang lebih canggih.

Ia menemukan empat struktur data utama di Python yang bisa mengatasi masalah ini: List, Tuple, Set, dan Dictionary.

List adalah struktur data yang paling serbaguna. List memungkinkan Brando untuk menyimpan banyak item dalam satu wadah dan item-item ini bisa diubah (mutable). Ia membayangkan list seperti sebuah rak panjang yang berisi banyak buku, dan ia bisa mengambil, mengganti, atau menambahkan buku kapan saja. 

# Membuat list untuk menyimpan nama-nama kue
daftar_kue = ["Kue Cokelat", "Kue Keju", "Kue Pandan", "Bolu Kukus"]

# Menambah item baru ke dalam list
daftar_kue.append("Donat")

# Mengakses item list berdasarkan indeks (dimulai dari 0)
kue_favorit = daftar_kue[0]
print(f"Kue favorit ibu adalah {kue_favorit}.")

# Mengubah item list
daftar_kue[3] = "Bolu Mekar"
print("Daftar kue terbaru:", daftar_kue)

Dengan list, Brando bisa dengan mudah mengelola daftar kue yang dijual ibunya. Ia bisa menambahkan kue baru, mengubah nama kue, atau menghapus kue yang sudah tidak dijual lagi.

Selanjutnya, ia belajar tentang Dictionary. Dictionary sangat cocok untuk menyimpan data dalam bentuk pasangan kunci-nilai (key-value). Brando membayangkan dictionary seperti buku telepon, di mana ia bisa mencari nomor telepon (nilai) dengan menggunakan nama orang (kunci). 

# Dictionary untuk menyimpan detail harga kue
harga_kue = {
    "Kue Cokelat": 12000,
    "Kue Keju": 15000,
    "Kue Pandan": 13500
}

# Mengakses nilai dengan menggunakan kunci
harga_cokelat = harga_kue["Kue Cokelat"]
print(f"Harga Kue Cokelat adalah {harga_cokelat} rupiah.")

# Menambah pasangan kunci-nilai baru
harga_kue["Donat"] = 5000
print("Harga kue terbaru:", harga_kue)

# Mengubah nilai dari sebuah kunci
harga_kue["Kue Keju"] = 16000
print("Harga Kue Keju yang diperbarui:", harga_kue["Kue Keju"])

Dictionary adalah solusi sempurna untuk menyimpan harga, stok, atau detail lainnya dari setiap jenis kue. Kunci (misalnya, "Kue Cokelat") sangat unik dan digunakan untuk mencari nilainya (misalnya, 12000).

Kemudian, Brando belajar tentang Tuple. Tuple sangat mirip dengan list, tetapi tidak bisa diubah (immutable). Sekali ia membuat tuple, isinya tidak bisa diganti, ditambahkan, atau dihapus. Brando menyadari tuple cocok untuk menyimpan data yang sifatnya permanen, seperti hari-hari dalam seminggu atau nama-nama bulan. 

# Tuple untuk menyimpan nama-nama hari
hari_kerja = ("Senin", "Selasa", "Rabu", "Kamis", "Jumat")

# Mencoba mengubah item dalam tuple akan menghasilkan error
# hari_kerja[0] = "Minggu" # Ini akan error!

print("Hari kerja ibu adalah:", hari_kerja)

Terakhir, Brando menemukan Set. Set adalah koleksi item yang tidak berurutan dan setiap itemnya unik. Tidak ada item yang bisa duplikat di dalam sebuah set. Set sangat berguna jika Brando ingin memastikan daftar kue yang ia miliki tidak ada yang kembar. 

# Set untuk menyimpan jenis-jenis bahan baku unik
bahan_baku = {"tepung", "gula", "telur", "cokelat", "gula", "telur"}

# Perhatikan bahwa "gula" dan "telur" hanya muncul satu kali
print("Bahan baku yang dibutuhkan:", bahan_baku)

# Menambah item baru
bahan_baku.add("keju")
print("Bahan baku terbaru:", bahan_baku)

Dengan pengetahuan tentang empat struktur data ini, Brando kini bisa mengelola data bisnis ibunya dengan sangat terstruktur. Ia bisa menggunakan list untuk menyimpan catatan penjualan harian, dictionary untuk menyimpan detail setiap kue, tuple untuk data yang tidak berubah, dan set untuk memastikan tidak ada data yang duplikat. Ia telah melangkah jauh dari hanya sekadar menggunakan variabel tunggal.
