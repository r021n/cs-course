"Komentar" – Catatan dalam Kode Python

Bayangkan Anda sedang menulis catatan di buku pelajaran Anda. Catatan itu bukan bagian dari materi pelajaran utama, tetapi Anda menuliskannya untuk membantu Anda memahami isi pelajaran, atau untuk mengingatkan diri sendiri tentang sesuatu. Dalam pemrograman Python, kita juga bisa melakukan hal yang sama dengan Komentar.

Komentar adalah baris atau blok teks dalam kode program yang diabaikan sepenuhnya oleh Python. Artinya, ketika Anda menjalankan program, Python akan melewati komentar-komentar ini tanpa memprosesnya. Komentar bukan perintah yang harus dijalankan; mereka ada di sana hanya untuk membantu manusia (Anda sendiri, atau teman Anda yang membaca kode) memahami apa yang sedang dilakukan oleh kode tersebut.

Menggunakan Tanda # untuk Komentar Satu Baris

Cara paling umum dan sederhana untuk menulis komentar dalam Python adalah dengan menggunakan tanda pagar (#). Apapun yang Anda tulis setelah tanda # pada baris yang sama akan dianggap sebagai komentar. 

# Ini adalah komentar satu baris. Python akan mengabaikan baris ini.
print("Halo siswa SMA!") # Komentar juga bisa diletakkan setelah kode.
print("Belajar Python itu menyenangkan.")
# Baris ini juga akan diabaikan oleh Python.

Outputnya:

Halo siswa SMA!
Belajar Python itu menyenangkan.

Penjelasan Kode:

Dalam contoh di atas, setiap baris yang dimulai dengan # adalah komentar. Anda bisa melihat bahwa meskipun ada banyak baris komentar, output program hanya menampilkan pesan dari perintah print(). Python sama sekali tidak menampilkan teks-teks yang dimulai dengan #. Bahkan ketika # diletakkan setelah perintah print("Halo siswa SMA!"), Python hanya menjalankan perintah print dan mengabaikan sisa baris setelah #.

Komentar "Multi-Baris": Catatan Panjang Tanpa Ribet

Bagaimana jika Anda ingin menulis komentar yang sangat panjang, lebih dari satu baris? Menambahkan # di setiap baris akan terasa merepotkan. Python tidak memiliki tanda khusus seperti # untuk komentar multi-baris. Namun, ada trik cerdas yang sering digunakan para programmer: yaitu dengan menggunakan tiga tanda kutip (baik ''' atau """).

Python menganggap teks yang diapit oleh tiga tanda kutip sebagai sebuah "teks biasa" (atau lebih tepatnya, "string"). Namun, jika teks panjang ini tidak disimpan ke dalam sebuah wadah (yang nanti akan kita pelajari sebagai "variabel"), maka Python akan melihatnya dan kemudian mengabaikannya. Ini seperti Anda menulis catatan di selembar kertas, tapi tidak Anda simpan ke dalam map, jadi Anda biarkan saja tergeletak dan tidak diproses lebih lanjut. 

'''
Ini adalah komentar yang mencakup beberapa baris.
Saya bisa menulis banyak hal di sini
untuk menjelaskan bagian kode yang kompleks atau
sekadar meninggalkan pesan penting.
Python akan mengabaikan semua teks di antara tiga tanda kutip ini.
'''
print("Selamat datang di dunia pemrograman!")

"""
Ini juga merupakan komentar multi-baris,
tapi menggunakan tiga tanda kutip ganda.
Hasilnya sama saja, Python tetap mengabaikannya.
"""
print("Semangat belajar!")

Outputnya:

Selamat datang di dunia pemrograman!
Semangat belajar!

Penjelasan Kode:

Di sini, blok teks yang diawali dan diakhiri dengan tiga tanda kutip (''' atau """) akan diabaikan oleh Python. Meskipun secara teknis Python mengenali teks tersebut sebagai "teks", karena teks itu tidak digunakan untuk tujuan lain (seperti ditampilkan atau disimpan), Python akan melewatinya begitu saja. Ini adalah cara yang sangat praktis untuk menulis penjelasan panjang, petunjuk, atau bahkan informasi hak cipta di awal program Anda tanpa perlu menambahkan # di setiap baris.

Mengapa Komentar Itu Sangat Penting?

Mungkin Anda berpikir, "Kalau Python mengabaikannya, untuk apa repot-repot menulis komentar?" Komentar memiliki beberapa peran yang sangat vital dalam menulis kode yang baik:

Untuk Diri Sendiri di Masa Depan: Pernahkah Anda menulis catatan untuk ujian, lalu beberapa bulan kemudian lupa maksud catatan itu? Kode juga sama. Komentar membantu Anda mengingat kembali mengapa Anda menulis bagian kode tertentu, terutama setelah lama tidak melihatnya. Ini sangat membantu saat Anda perlu memperbarui atau memperbaiki program Anda di kemudian hari.

Untuk Orang Lain (dan Tim): Jika Anda bekerja sama dengan teman atau orang lain dalam membuat program, komentar adalah "penerjemah" kode Anda. Mereka bisa dengan cepat memahami tujuan dan cara kerja bagian-bagian program tanpa harus menebak-nebak. Ini mempercepat kolaborasi dan mengurangi kesalahpahaman.

Sebagai "Tombol Nonaktif" Sementara: Komentar bisa menjadi alat yang ampuh saat Anda sedang mencoba-coba atau memperbaiki program. Jika ada bagian kode yang Anda curigai menyebabkan masalah, atau Anda hanya ingin melihat bagaimana program berjalan tanpa bagian itu, Anda bisa mengubahnya menjadi komentar. 

# Ini adalah kode yang menghitung sesuatu (nanti kita akan pelajari)
# hasil_hitung = 10 * 5
# print("Hasil perhitungan:", hasil_hitung)

print("Program berjalan dengan baik tanpa bagian yang dikomentari.")

# print("Baris ini dinonaktifkan sementara.")
print("Ini adalah baris penting yang harus selalu aktif.")

Outputnya:

Program berjalan dengan baik tanpa bagian yang dikomentari.
Ini adalah baris penting yang harus selalu aktif.

Penjelasan Kode:

Dalam contoh ini, baris hasil_hitung = 10 * 5 dan print("Hasil perhitungan:", hasil_hitung) diubah menjadi komentar. Ini berarti Python tidak akan menjalankannya sama sekali. Bayangkan jika Anda sedang mencari tahu di mana letak kesalahan di program Anda. Daripada menghapus baris kode yang mungkin penting, Anda bisa mengubahnya menjadi komentar. Jika ternyata bukan itu masalahnya, Anda tinggal menghapus tanda # untuk mengaktifkannya kembali. Ini adalah teknik yang sangat umum dan efektif dalam proses pengembangan program.

Dengan memahami dan menggunakan komentar secara bijak, Anda tidak hanya membuat kode yang berfungsi, tetapi juga kode yang "ramah" bagi manusia!
