"Class" – Cetak Biru untuk Membuat Objek di Python

Sejauh ini, kita telah belajar bagaimana menyimpan data dalam variabel, melakukan perhitungan, membuat keputusan dengan kondisional, mengulang tugas dengan perulangan, dan mengelompokkan kode dengan fungsi. Semua ini adalah dasar yang sangat kuat. Namun, ketika program menjadi lebih besar dan kompleks, kita sering berurusan dengan "benda-benda" yang memiliki ciri-ciri (data) dan kemampuan (fungsi) tertentu.

Bayangkan Anda ingin membuat program untuk mengelola data siswa di sekolah. Setiap siswa memiliki nama, usia, kelas, dan mungkin nilai. Setiap siswa juga bisa "melakukan" sesuatu, seperti "belajar", atau "mengikuti ujian". Jika Anda menyimpan semua informasi ini di variabel terpisah (nama_siswa1, usia_siswa1, nama_siswa2, usia_siswa2), ini akan sangat berantakan.

Di sinilah Class (dibaca "klas") muncul. Class adalah sebuah cetak biru, template, atau definisi untuk membuat "benda-benda" (yang kita sebut objek) yang memiliki sifat dan perilaku serupa. Class mendefinisikan:

  1. Atribut: Ini adalah data atau ciri-ciri yang akan dimiliki oleh setiap objek yang dibuat dari Class ini. Contoh: nama, usia, kelas.
  2. Metode: Ini adalah fungsi atau kemampuan yang bisa dilakukan oleh setiap objek yang dibuat dari Class ini. Contoh: belajar(), mengikuti_ujian().

Jadi, daripada membuat variabel terpisah untuk setiap siswa, Anda bisa membuat satu Class Siswa. Dari Class Siswa ini, Anda bisa membuat banyak objek siswa (misalnya, objek_budi, objek_siti), dan setiap objek akan memiliki atribut dan metode sendiri sesuai cetak birunya.

1. Membuat Class Sederhana: Class Siswa

Kita membuat Class menggunakan kata kunci class, diikuti nama Class, dan diakhiri dengan titik dua (:). Biasanya, nama Class diawali dengan huruf kapital (ini adalah konvensi PEP 8). 

# Ini adalah definisi (cetak biru) dari Class Siswa
class Siswa:
    # Ini adalah metode (fungsi) yang dimiliki oleh setiap objek Siswa
    def belajar(self, mata_pelajaran):
        print(f"Siswa ini sedang belajar {mata_pelajaran}.")

    def sapa(self):
        print("Halo! Saya adalah seorang siswa.")

# Membuat objek (instansi) dari Class Siswa
# Ini seperti membuat sebuah 'siswa' nyata dari cetak biru 'Class Siswa'
siswa_budi = Siswa()
siswa_siti = Siswa()

print("--- Interaksi dengan Objek Siswa ---")
siswa_budi.sapa()        # Memanggil metode sapa() pada objek siswa_budi
siswa_budi.belajar("Matematika") # Memanggil metode belajar()

siswa_siti.sapa()
siswa_siti.belajar("Bahasa Inggris")

Outputnya:

--- Interaksi dengan Objek Siswa ---
Halo! Saya adalah seorang siswa.
Siswa ini sedang belajar Matematika.
Halo! Saya adalah seorang siswa.
Siswa ini sedang belajar Bahasa Inggris.

Penjelasan Kode:

  1. class Siswa:: Baris ini mendefinisikan sebuah Class bernama Siswa.
  2. def belajar(self, mata_pelajaran): dan def sapa(self):: Ini adalah metode dari Class Siswa. Perhatikan parameter self yang ada di setiap metode. self ini adalah referensi ke objek itu sendiri yang sedang memanggil metode tersebut. Python secara otomatis memberikan objek yang sedang aktif sebagai argumen pertama ke self.
  3. siswa_budi = Siswa() dan siswa_siti = Siswa(): Ini adalah proses instansiasi, yaitu membuat objek nyata dari Class Siswa. siswa_budi dan siswa_siti adalah dua objek yang berbeda, meskipun mereka dibuat dari cetak biru yang sama.
  4. siswa_budi.sapa(): Ini adalah cara memanggil metode sapa() yang "dimiliki" oleh objek siswa_budi.
  5. Saat siswa_budi.belajar("Matematika") dipanggil, di dalam metode belajar, self akan merujuk ke siswa_budi, dan mata_pelajaran akan berisi "Matematika".

2. Atribut: Data Unik Setiap Objek dengan Metode Khusus __init__

Objek-objek biasanya memiliki data yang unik untuk dirinya sendiri. Misalnya, siswa_budi punya nama "Budi" dan siswa_siti punya nama "Siti". Kita menyimpan data unik ini di atribut instansi (instance attributes).

Untuk mengatur atribut instansi ini saat objek pertama kali dibuat, kita menggunakan metode khusus bernama __init__ (dibaca "dunder init"). Metode ini secara otomatis dipanggil oleh Python setiap kali Anda membuat objek baru dari sebuah Class. Ini seperti "fungsi persiapan" objek. 

class SiswaBaru:
    def __init__(self, nama, umur, kelas):
        # self.nama, self.umur, self.kelas adalah atribut instansi
        # Mereka menyimpan data unik untuk SETIAP objek SiswaBaru
        self.nama = nama
        self.umur = umur
        self.kelas = kelas
        print(f"Objek SiswaBaru '{self.nama}' berhasil dibuat!")

    def tampilkan_biodata(self):
        # Metode ini mengakses atribut instansi dari objek itu sendiri
        print(f"Nama: {self.nama}")
        print(f"Umur: {self.umur} tahun")
        print(f"Kelas: {self.kelas}")

    def ulang_tahun(self):
        self.umur += 1 # Mengubah atribut umur dari objek itu sendiri
        print(f"Selamat ulang tahun, {self.nama}! Anda sekarang berusia {self.umur} tahun.")

# Membuat objek SiswaBaru (otomatis memanggil __init__)
siswa_adi = SiswaBaru("Adi", 15, "X MIPA 2") # Memberikan nilai untuk nama, umur, kelas
siswa_luna = SiswaBaru("Luna", 16, "XI IPS 1")

print("\n--- Biodata Siswa ---")
siswa_adi.tampilkan_biodata()
print("-" * 20)
siswa_luna.tampilkan_biodata()

print("\n--- Ulang Tahun Adi ---")
siswa_adi.ulang_tahun() # Adi merayakan ulang tahun!
print("-" * 20)
siswa_adi.tampilkan_biodata() # Umur Adi sudah berubah
siswa_luna.tampilkan_biodata() # Umur Luna tidak berubah

Outputnya:

Objek SiswaBaru 'Adi' berhasil dibuat!
Objek SiswaBaru 'Luna' berhasil dibuat!

--- Biodata Siswa ---
Nama: Adi
Umur: 15 tahun
Kelas: X MIPA 2
--------------------
Nama: Luna
Umur: 16 tahun
Kelas: XI IPS 1

--- Ulang Tahun Adi ---
Selamat ulang tahun, Adi! Anda sekarang berusia 16 tahun.
--------------------
Nama: Adi
Umur: 16 tahun
Kelas: X MIPA 2
Nama: Luna
Umur: 16 tahun
Kelas: XI IPS 1

Penjelasan Kode:

  1. def __init__(self, nama, umur, kelas):: Ini adalah metode khusus yang dipanggil saat objek dibuat.
    • self: Selalu parameter pertama, merujuk pada objek yang sedang dibuat.
    • nama, umur, kelas: Ini adalah parameter yang harus Anda berikan saat membuat objek (SiswaBaru("Adi", 15, "X MIPA 2")).
  2. self.nama = nama, self.umur = umur, self.kelas = kelas: Ini adalah baris-baris yang membuat atribut instansi. self.nama artinya "atribut nama untuk objek ini". Nilai dari parameter nama akan disimpan di atribut self.nama pada objek yang sedang dibuat.
  3. siswa_adi.tampilkan_biodata(): Metode ini menggunakan self.nama, self.umur, dan self.kelas yang sudah disimpan di objek siswa_adi.
  4. siswa_adi.ulang_tahun(): Metode ini mengubah atribut self.umur dari objek siswa_adi. Perhatikan bahwa siswa_luna tidak terpengaruh, karena atribut umur mereka adalah unik untuk setiap objek.

3. Atribut Kelas: Informasi yang Dibagi Bersama

Selain atribut instansi yang unik untuk setiap objek, Class juga bisa memiliki atribut kelas. Ini adalah data yang dibagi oleh semua objek yang dibuat dari Class tersebut. Mereka adalah bagian dari Class itu sendiri, bukan bagian dari objek individu. 

class Sekolah:
    # Ini adalah atribut kelas (shared by all objects of this class)
    # Ini adalah variabel yang dimiliki oleh Class 'Sekolah' itu sendiri, bukan objeknya
    NAMA_SEKOLAH = "SMA Bintang Harapan"
    JUMLAH_MAKSIMAL_SISWA = 1000

    def __init__(self, nama_kepsek, alamat):
        self.nama_kepsek = nama_kepsek
        self.alamat = alamat
        print(f"Objek Sekolah '{self.NAMA_SEKOLAH}' dibentuk dengan Kepsek {self.nama_kepsek}.")

    def info_sekolah(self):
        print(f"Sekolah: {self.NAMA_SEKOLAH}") # Akses atribut kelas melalui self
        print(f"Alamat: {self.alamat}")
        print(f"Kepala Sekolah: {self.nama_kepsek}")
        print(f"Kapasitas Maksimal: {self.JUMLAH_MAKSIMAL_SISWA} siswa.")

# Membuat objek dari Class Sekolah
sekolah_pusat = Sekolah("Bpk. Ahmad", "Jl. Utama No. 1")
sekolah_cabang = Sekolah("Ibu Dewi", "Jl. Cabang No. 5")

print("\n--- Informasi Sekolah ---")
sekolah_pusat.info_sekolah()
print("-" * 30)
sekolah_cabang.info_sekolah()

print("\n--- Mengakses Atribut Kelas Secara Langsung ---")
# Anda bisa mengakses atribut kelas langsung dari Class-nya
print(f"Nama Sekolah dari Class: {Sekolah.NAMA_SEKOLAH}")
print(f"Kapasitas Maksimal dari Class: {Sekolah.JUMLAH_MAKSIMAL_SISWA}")

# Jika Anda mengubah atribut kelas, semua objek akan melihat perubahan itu
Sekolah.NAMA_SEKOLAH = "SMA Bintang Jaya"
print("\n--- Nama Sekolah Berubah ---")
sekolah_pusat.info_sekolah() # Objek pusat melihat nama yang baru
sekolah_cabang.info_sekolah() # Objek cabang juga melihat nama yang baru

Penjelasan Kode:

  1. NAMA_SEKOLAH = "SMA Bintang Harapan" dan JUMLAH_MAKSIMAL_SISWA = 1000: Ini adalah atribut kelas, didefinisikan langsung di bawah class Sekolah:, di luar metode manapun.
  2. Kita bisa mengakses atribut kelas ini baik melalui objek (self.NAMA_SEKOLAH) maupun langsung melalui nama Class (Sekolah.NAMA_SEKOLAH).
  3. Yang penting: ketika Sekolah.NAMA_SEKOLAH = "SMA Bintang Jaya" dijalankan, perubahan ini akan terlihat oleh semua objek dari Class Sekolah, karena mereka semua berbagi atribut kelas yang sama. Ini berbeda dengan atribut instansi yang unik untuk setiap objek.

4. Hal yang Perlu Diperhatikan: Class Itu Sendiri Adalah Objek

Di Python, Class itu sendiri adalah sebuah "benda" (objek) yang bisa memiliki atribut dan perilaku. Ketika Anda mendefinisikan sebuah Class, Anda sebenarnya sedang membuat sebuah objek Class di memori Python. 

class ContohObjekClass:
    # Atribut kelas
    jumlah_dibuat = 0

    def __init__(self, nama):
        self.nama = nama
        ContohObjekClass.jumlah_dibuat += 1 # Menambah hitungan atribut kelas

    def tampilkan_nama(self):
        print(f"Nama objek ini: {self.nama}")

# Sebelum membuat objek, Class ContohObjekClass sudah ada
print(f"Tipe dari ContohObjekClass: {type(ContohObjekClass)}")
print(f"Jumlah objek yang dibuat (sebelum ada): {ContohObjekClass.jumlah_dibuat}")

obj1 = ContohObjekClass("Objek Pertama")
obj2 = ContohObjekClass("Objek Kedua")

print(f"Jumlah objek yang dibuat (setelah 2 objek): {ContohObjekClass.jumlah_dibuat}")

# Class juga bisa memiliki atribut yang ditambahkan secara dinamis (tidak disarankan)
ContohObjekClass.versi = "1.0"
print(f"Versi Class: {ContohObjekClass.versi}")

Outputnya:

Tipe dari ContohObjekClass: <class 'type'>
Jumlah objek yang dibuat (sebelum ada): 0
Jumlah objek yang dibuat (setelah 2 objek): 2
Versi Class: 1.0

Penjelasan Kode:

  1. type(ContohObjekClass) menunjukkan bahwa ContohObjekClass itu sendiri adalah objek dari tipe type. Ini konsep yang lebih dalam tapi intinya, Class adalah entitas nyata di Python.
  2. ContohObjekClass.jumlah_dibuat adalah atribut kelas yang bisa diakses dan dimodifikasi langsung melalui nama Class. Setiap kali objek baru dibuat, __init__ menambah nilai jumlah_dibuat ini dengan 1, dan semua objek "melihat" nilai yang sama.
  3. ContohObjekClass.versi = "1.0" menunjukkan bahwa Anda bahkan bisa menambahkan atribut ke Class itu sendiri setelah definisinya. Ini mirip dengan menambahkan atribut ke objek biasa. Namun, praktik ini jarang dilakukan dan biasanya atribut kelas didefinisikan di dalam blok class.

5. Hal yang Perlu Diperhatikan: Atribut Bisa Ditambah di Luar __init__ (Tapi Sebaiknya Tidak)

Python adalah bahasa yang sangat fleksibel. Anda bisa menambahkan atribut baru ke sebuah objek setelah objek itu dibuat, bahkan jika atribut tersebut tidak didefinisikan di __init__. 

class Karyawan:
    def __init__(self, nama):
        self.nama = nama # Atribut 'nama' dibuat di __init__

    def info(self):
        print(f"Nama Karyawan: {self.nama}")
        # Mencoba mengakses atribut yang mungkin tidak ada
        if hasattr(self, 'divisi'): # Fungsi bawaan untuk cek apakah objek punya atribut
            print(f"Divisi: {self.divisi}")
        else:
            print("Divisi: Belum ditetapkan")

# Membuat objek
karyawan1 = Karyawan("Budi")
karyawan2 = Karyawan("Siti")

print("--- Sebelum Tambah Atribut ---")
karyawan1.info()
karyawan2.info()

# Menambahkan atribut 'divisi' ke karyawan1 SAJA (setelah objek dibuat)
karyawan1.divisi = "Pemasaran"

print("\n--- Setelah Tambah Atribut ke Karyawan1 ---")
karyawan1.info()
karyawan2.info() # Karyawan2 tidak punya atribut 'divisi' ini

# print(karyawan2.divisi) # Ini akan error! AttributeError karena karyawan2 tidak punya 'divisi'

Outputnya:

--- Sebelum Tambah Atribut ---
Nama Karyawan: Budi
Divisi: Belum ditetapkan
Nama Karyawan: Siti
Divisi: Belum ditetapkan

--- Setelah Tambah Atribut ke Karyawan1 ---
Nama Karyawan: Budi
Divisi: Pemasaran
Nama Karyawan: Siti
Divisi: Belum ditetapkan

Penjelasan Kode:

Pada karyawan1.divisi = "Pemasaran", kita menambahkan atribut baru bernama divisi langsung ke objek karyawan1. Atribut ini hanya ada di karyawan1 dan tidak di karyawan2.

Meskipun Python mengizinkan ini, ini umumnya bukan praktik yang disarankan dalam Class yang dirancang dengan baik. Atribut sebaiknya didefinisikan di __init__ agar semua objek dari Class yang sama memiliki set atribut yang konsisten. Menambahkan atribut secara dinamis bisa membuat kode Anda sulit diprediksi dan rawan kesalahan. Gunakan hasattr() untuk memeriksa keberadaan atribut jika Anda memang perlu menangani kasus atribut yang mungkin tidak ada.

Kapan Menggunakan Class?

Anda akan menggunakan Class ketika:

  1. Anda ingin merepresentasikan "benda" atau "entitas" nyata yang memiliki data (atribut) dan perilaku (metode). Contoh: Siswa, Buku, Mobil, Akun Bank, Produk.
  2. Anda ingin mengelompokkan data dan fungsi yang relevan secara bersamaan. Class membantu menjaga kode tetap terorganisir.
  3. Anda perlu membuat banyak objek yang memiliki struktur dan perilaku yang sama, tetapi dengan data yang berbeda (misalnya, banyak objek Siswa yang masing-masing punya nama, umur, dan kelas yang berbeda).
  4. Anda ingin membuat kode yang lebih modular dan mudah digunakan kembali.

Class adalah fondasi dari paradigma pemrograman yang disebut Object-Oriented Programming (OOP), yang akan kita bahas lebih lanjut di modul berikutnya. Memahami Class adalah langkah pertama untuk menulis program yang lebih besar, terstruktur, dan efisien.
