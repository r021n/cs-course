"File Handling" – Menyimpan dan Membaca Data dari File

Sejauh ini, semua data yang kita olah di program Python hanya bersifat sementara. Ketika program selesai berjalan, semua variabel dan informasinya akan hilang. Lalu, bagaimana jika kita ingin menyimpan data agar bisa digunakan lagi nanti, bahkan setelah program ditutup dan dibuka kembali? Di sinilah konsep File Handling (Penanganan File) berperan.

File Handling adalah kemampuan program untuk berinterinteraksi dengan file yang ada di komputer Anda. Anda bisa:

  • Membaca isi file (misalnya, membaca catatan harian atau data dari sensor).
  • Menulis data baru ke dalam file (misalnya, menyimpan hasil perhitungan atau catatan pengguna).
  • Menambah data ke file yang sudah ada (misalnya, menambahkan entri baru ke log).

Python menyediakan fungsi bawaan bernama open() untuk membuka file, dan metode-metode lain untuk membaca atau menulisnya.

1. Membuka File: Fungsi open()

Sebelum Anda bisa membaca atau menulis ke sebuah file, Anda harus "membuka" file tersebut. Fungsi open() digunakan untuk tugas ini. Ia mengembalikan sebuah "objek file" yang akan kita gunakan untuk berinteraksi dengan file.

Sintaks dasar: file_object = open("nama_file.txt", "mode")

  • "nama_file.txt": Nama file (termasuk lokasi folder jika file tidak di folder yang sama dengan program Anda).
  • "mode": Mode untuk membuka file. Mode yang paling umum:
    • "r" (read): Mode hanya-baca. Anda hanya bisa membaca isi file. Jika file tidak ada, akan menyebabkan error.
    • "w" (write): Mode tulis. Ini akan membuat file baru jika file belum ada. Jika file sudah ada, isinya akan dihapus sepenuhnya dan diganti dengan data baru yang Anda tulis. Hati-hati dengan mode ini!
    • "a" (append): Mode tambah. Ini akan menambahkan data ke akhir file. Jika file belum ada, ia akan membuat file baru.
    • "x" (create): Mode buat baru. Ini akan membuat file baru. Jika file sudah ada, akan menyebabkan error.
    • "t" (text): Mode teks (default). Membuka file sebagai teks.
    • "b" (binary): Mode biner. Membuka file sebagai data biner (misalnya gambar, video).
    (Anda bisa menggabungkan mode, misal "rt" untuk "read text", "wb" untuk "write binary") 
# Membuat file baru dan menulis ke dalamnya (mode 'w')
# Pastikan tidak ada file 'catatan.txt' di folder yang sama, atau isinya akan terhapus!
file_tulis = open("catatan.txt", "w")
file_tulis.write("Ini adalah baris pertama dari catatan saya.\n")
file_tulis.write("Ini baris kedua.\n")
file_tulis.close() # Sangat penting untuk menutup file!
print("File 'catatan.txt' telah dibuat dan diisi.")

# Membaca isi file (mode 'r')
file_baca = open("catatan.txt", "r")
isi_file = file_baca.read() # Membaca seluruh isi file
print("\n--- Isi dari catatan.txt ---")
print(isi_file)
file_baca.close() # Menutup file setelah selesai membaca

# Menambahkan ke file yang sudah ada (mode 'a')
file_tambah = open("catatan.txt", "a")
file_tambah.write("Ini adalah baris tambahan di akhir.\n")
file_tambah.close()
print("\nFile 'catatan.txt' telah ditambahkan.")

# Baca lagi untuk melihat perubahannya
file_baca_lagi = open("catatan.txt", "r")
print("\n--- Isi catatan.txt setelah ditambahkan ---")
print(file_baca_lagi.read())
file_baca_lagi.close()

Outputnya:

File 'catatan.txt' telah dibuat dan diisi.

--- Isi dari catatan.txt ---
Ini adalah baris pertama dari catatan saya.
Ini baris kedua.

File 'catatan.txt' telah ditambahkan.

--- Isi catatan.txt setelah ditambahkan ---
Ini adalah baris pertama dari catatan saya.
Ini baris kedua.
Ini adalah baris tambahan di akhir.

Penjelasan Kode:

  1. file_tulis = open("catatan.txt", "w"): Baris ini mencoba membuka catatan.txt dalam mode tulis ("w"). Jika file tidak ada, ia akan dibuat. Jika ada, isinya akan dihapus.
  2. file_tulis.write("..."): Metode .write() digunakan untuk menulis teks ke file. \n digunakan untuk membuat baris baru.
  3. file_tulis.close(): Ini adalah langkah yang sangat krusial! Anda harus selalu menutup file setelah selesai membaca atau menulis. Jika tidak, data mungkin tidak tersimpan dengan benar, atau file mungkin tetap "terkunci" oleh program Anda sehingga program lain tidak bisa mengaksesnya.
  4. Langkah-langkah berikutnya menunjukkan membaca ("r") dan menambahkan ("a") ke file yang sama.

2. Menutup File Secara Otomatis: Pernyataan with open() (Sangat Disarankan!)

Meskipun file.close() itu penting, terkadang kita lupa melakukannya, atau program mungkin mengalami error di tengah jalan sehingga close() tidak sempat dipanggil. Ini bisa menyebabkan masalah.

Python memiliki cara yang lebih aman dan direkomendasikan untuk bekerja dengan file, yaitu menggunakan pernyataan with open(...) as ...: (disebut "context manager"). Keuntungan utamanya adalah: file akan secara otomatis ditutup begitu program keluar dari blok with, bahkan jika terjadi error! 

# Menulis ke file dengan 'with open()'
print("--- Menulis dengan with open() ---")
with open("daftar_siswa.txt", "w") as file_siswa:
    file_siswa.write("Nama: Andi\n")
    file_siswa.write("Kelas: X IPA 1\n")
    # Jika ada error di sini, file tetap akan ditutup

print("File 'daftar_siswa.txt' telah ditulis dengan aman.")

# Membaca dari file dengan 'with open()'
print("\n--- Membaca dengan with open() ---")
with open("daftar_siswa.txt", "r") as file_siswa:
    isi_lengkap = file_siswa.read()
    print(isi_lengkap)

print("File 'daftar_siswa.txt' telah dibaca dengan aman.")

Outputnya:

--- Menulis dengan with open() ---
File 'daftar_siswa.txt' telah ditulis dengan aman.

--- Membaca dengan with open() ---
Nama: Andi
Kelas: X IPA 1

File 'daftar_siswa.txt' telah dibaca dengan aman.

Penjelasan Kode:

Pola with open("nama_file.txt", "mode") as nama_variabel_file: sangat direkomendasikan. Python menjamin bahwa file_siswa.close() akan dipanggil secara otomatis begitu kode keluar dari blok yang menjorok di bawah with, tidak peduli apa yang terjadi di dalamnya. Ini adalah cara yang lebih "tahan banting" dalam bekerja dengan file.

3. Membaca File: Berbagai Cara

Selain read() yang membaca seluruh isi file, ada cara lain untuk membaca file:

  • readline(): Membaca satu baris teks saja. Setiap kali dipanggil, ia akan membaca baris berikutnya.
  • readlines(): Membaca semua baris dan mengembalikannya sebagai sebuah List, di mana setiap elemen List adalah satu baris dari file (termasuk karakter \n di akhir baris).
  • Melakukan perulangan (for loop) pada objek file: Ini adalah cara paling umum dan efisien untuk membaca file baris demi baris. 
# Siapkan file untuk contoh pembacaan
with open("pesan_rahasia.txt", "w") as f:
    f.write("Baris 1: Ini adalah pesan rahasia.\n")
    f.write("Baris 2: Jangan sampai bocor.\n")
    f.write("Baris 3: Rahasia antara kita saja.\n")

print("File 'pesan_rahasia.txt' telah dibuat.")

# Membaca satu per satu baris dengan readline()
print("\n--- Membaca dengan readline() ---")
with open("pesan_rahasia.txt", "r") as f:
    baris_pertama = f.readline()
    baris_kedua = f.readline()
    print(f"Baris 1: {baris_pertama.strip()}") # strip() untuk menghilangkan \n
    print(f"Baris 2: {baris_kedua.strip()}")

# Membaca semua baris ke dalam List dengan readlines()
print("\n--- Membaca dengan readlines() ---")
with open("pesan_rahasia.txt", "r") as f:
    semua_baris_list = f.readlines()
    print("Isi file dalam bentuk list:")
    for baris in semua_baris_list:
        print(baris.strip()) # strip() setiap baris saat mencetak

# Membaca baris demi baris dengan perulangan 'for' (paling umum dan efisien)
print("\n--- Membaca dengan perulangan for ---")
with open("pesan_rahasia.txt", "r") as f:
    for nomor_baris, baris in enumerate(f): # enumerate() memberi nomor indeks
        print(f"Baris {nomor_baris + 1}: {baris.strip()}")

Outputnya:

File 'pesan_rahasia.txt' telah dibuat.

--- Membaca dengan readline() ---
Baris 1: Baris 1: Ini adalah pesan rahasia.
Baris 2: Baris 2: Jangan sampai bocor.

--- Membaca dengan readlines() ---
Isi file dalam bentuk list:
Baris 1: Ini adalah pesan rahasia.
Baris 2: Jangan sampai bocor.
Baris 3: Rahasia antara kita saja.

--- Membaca dengan perulangan for ---
Baris 1: Baris 1: Ini adalah pesan rahasia.
Baris 2: Baris 2: Jangan sampai bocor.
Baris 3: Rahasia antara kita saja.

Penjelasan Kode:

  • readline(): Membaca satu baris. Kursor file akan bergeser ke baris berikutnya.
  • readlines(): Mengembalikan list. Setiap elemen list adalah satu baris lengkap dari file, termasuk karakter \n (newline) di akhir. Anda seringkali perlu menggunakan .strip() pada setiap baris untuk menghilangkan karakter \n dan spasi kosong di awal/akhir.
  • Perulangan for (pada objek file langsung): Ini adalah cara yang paling direkomendasikan untuk membaca file besar baris demi baris. Python secara otomatis mengulang setiap baris dalam file, dan ini sangat efisien karena tidak memuat seluruh file ke memori sekaligus (seperti readlines() bisa lakukan pada file yang sangat besar). enumerate() adalah fungsi bawaan yang berguna untuk mendapatkan indeks (nomor urut) saat mengulang.

4. Menulis ke File: write() dan writelines()

  • write(string): Menulis satu string ke file. Ingat, Anda harus menambahkan \n secara manual jika ingin membuat baris baru.
  • writelines(list_of_strings): Menulis semua string dari sebuah list ke file. Setiap string di list ditulis secara berurutan. Anda tetap harus memastikan setiap string di list diakhiri dengan \n jika Anda ingin setiap item menjadi baris terpisah di file. 
# Menulis data ke file siswa_baru.txt
nama_siswa_baru = ["Adi Purnomo\n", "Bela Cantika\n", "Citra Dewi\n"] # Perhatikan \n di setiap nama!

print("--- Menulis dengan writelines() ---")
with open("siswa_baru.txt", "w") as f_siswa:
    f_siswa.writelines(nama_siswa_baru)
print("Data siswa baru telah ditulis.")

# Membaca kembali untuk verifikasi
with open("siswa_baru.txt", "r") as f_siswa:
    print("\nIsi siswa_baru.txt:")
    print(f_siswa.read())

# Contoh: Menulis list angka ke file (perlu konversi ke string)
nilai_data = [95, 88, 72, 91, 65]
with open("nilai_output.txt", "w") as f_nilai:
    for nilai in nilai_data:
        f_nilai.write(str(nilai) + "\n") # Konversi angka ke string dan tambahkan baris baru
print("\nNilai telah ditulis ke nilai_output.txt")

Outputnya:

--- Menulis dengan writelines() ---
Data siswa baru telah ditulis.

Isi siswa_baru.txt:
Adi Purnomo
Bela Cantika
Citra Dewi

Nilai telah ditulis ke nilai_output.txt

Penjelasan Kode:

writelines() berguna jika Anda sudah punya list string yang siap ditulis. Namun, penting untuk diingat bahwa writelines() tidak secara otomatis menambahkan \n. Jadi, jika Anda ingin setiap item list menjadi baris terpisah di file, Anda harus menambahkan \n di akhir setiap string di dalam list tersebut.

Ketika menulis angka, Anda harus selalu mengonversinya menjadi string terlebih dahulu menggunakan str() sebelum menuliskannya ke file teks, seperti f_nilai.write(str(nilai) + "\n").

5. Penanganan Error Saat File Handling (Penting!)

Membaca dan menulis file adalah operasi yang rentan error. Misalnya, file mungkin tidak ditemukan, atau Anda mungkin tidak memiliki izin untuk menulis ke suatu lokasi. Inilah saatnya konsep try-except yang sudah kita pelajari menjadi sangat penting! 

# Contoh: Menangani error saat membuka file
print("--- Penanganan Error File Handling ---")
try:
    # Coba membuka file yang tidak ada (mode 'r')
    with open("file_yang_tidak_ada.txt", "r") as f:
        print(f.read())
except FileNotFoundError:
    print("Error: File tidak ditemukan. Pastikan nama file dan lokasi sudah benar.")
except PermissionError:
    print("Error: Anda tidak memiliki izin untuk mengakses file ini.")
except Exception as e:
    print(f"Terjadi error lain saat membaca file: {type(e).__name__}: {e}")

print("\n")

# Contoh: Menangani error saat menulis file
try:
    # Coba menulis ke lokasi yang mungkin tidak ada izin (misal: C:/ tanpa admin di Windows)
    # Ini mungkin tidak selalu error tergantung OS dan izin Anda
    with open("/lokasi/yang/invalid/file_baru.txt", "w") as f: # Contoh lokasi invalid
        f.write("Mencoba menulis ke lokasi terlarang.")
except FileNotFoundError: # Juga bisa FileExistsError jika folder tidak ada
    print("Error: Lokasi folder tidak ditemukan atau tidak valid.")
except PermissionError:
    print("Error: Anda tidak memiliki izin menulis di lokasi ini. Coba jalankan sebagai Administrator.")
except Exception as e:
    print(f"Terjadi error tak terduga saat menulis: {type(e).__name__}: {e}")

Outputnya (jika file_yang_tidak_ada.txt memang tidak ada): 

--- Penanganan Error File Handling ---
Error: File tidak ditemukan. Pastikan nama file dan lokasi sudah benar.

Outputnya (jika '/lokasi/yang/invalid/file_baru.txt' tidak ada izin):

(Output ini akan sangat bervariasi tergantung sistem operasi dan izin pengguna. Bisa jadi FileNotFoundError, PermissionError, atau bahkan OSError.) 

Error: Lokasi folder tidak ditemukan atau tidak valid.

Penjelasan Kode:

  • Ketika kita mencoba open("file_yang_tidak_ada.txt", "r"), jika file itu memang tidak ada, Python akan memicu FileNotFoundError. Blok except FileNotFoundError akan menangkapnya, dan kita bisa mencetak pesan yang lebih informatif daripada stack trace yang membingungkan.
  • Error seperti PermissionError (jika program tidak punya izin membaca/menulis) atau OSError (kesalahan sistem operasi umum) juga bisa terjadi. Selalu bagus untuk mengantisipasi error yang paling umum terjadi saat berinteraksi dengan sistem luar seperti file.

Hal yang Harus Diperhatikan: Path Relatif vs. Absolut

Ketika Anda memberikan nama file seperti "catatan.txt", itu adalah jalur relatif (relative path). Python akan mencari file itu di direktori yang sama dengan tempat Anda menjalankan program Python.

  • Jika program Anda ada di C:\Users\Saya\ProyekPython\main.py
  • Dan Anda menjalankan python main.py dari C:\Users\Saya\ProyekPython\
  • Maka open("catatan.txt", "w") akan membuat file di lokasi C:\Users\Saya\ProyekPython\catatan.txt.

Namun, jika Anda ingin mengakses file di lokasi yang spesifik di komputer Anda, Anda perlu menggunakan jalur absolut (absolute path).

  • Windows: C:\\Users\\NamaAnda\\Dokumen\\data.txt (perhatikan dua backslash \\ atau gunakan forward slash / seperti di Linux/macOS: C:/Users/NamaAnda/Dokumen/data.txt).
  • Linux/macOS: /home/namaanda/dokumen/data.txt

Menggunakan jalur yang benar sangat penting. Untuk keamanan dan kemudahan, biasanya disarankan untuk menyimpan file data di sub-folder yang sama dengan program atau folder yang sudah ditentukan untuk data aplikasi.

Dengan modul "File Handling" ini, Anda sekarang memiliki kemampuan untuk membuat program Anda berinteraksi dengan sistem file komputer. Anda bisa menyimpan data secara permanen, membaca konfigurasi, atau mencatat aktivitas program. Ini adalah langkah maju yang signifikan dalam membangun aplikasi yang lebih fungsional!
