"Fungsi" – Membuat "Alat Bantu" dalam Program Python

Bayangkan Anda memiliki sebuah resep masakan favorit. Daripada menulis ulang semua langkah resep setiap kali Anda ingin memasak hidangan itu, Anda cukup menulis nama resepnya, dan semua orang tahu langkah-langkah apa yang harus dilakukan. Atau, bayangkan sebuah kalkulator. Anda cukup menekan tombol + atau x, dan kalkulator akan melakukan serangkaian perhitungan rumit di balik layar untuk memberikan hasil.

Dalam pemrograman Python, Fungsi adalah persis seperti itu. Fungsi adalah sebuah blok kode yang diberi nama, yang melakukan tugas tertentu. Anda bisa "memanggil" atau "menggunakan" fungsi ini berkali-kali di berbagai bagian program Anda tanpa perlu menulis ulang semua kode di dalamnya. Fungsi membantu kita membuat kode menjadi lebih rapi, terstruktur, dan mudah digunakan kembali.

Mengapa Kita Butuh Fungsi? (Prinsip DRY - Don't Repeat Yourself)

Ada beberapa alasan kuat mengapa fungsi itu sangat penting:

  1. Mengurangi Pengulangan Kode: Jika ada beberapa bagian kode yang melakukan tugas yang sama, kita bisa mengubahnya menjadi fungsi. Daripada menulis ulang kode yang sama berkali-kali, kita cukup memanggil fungsi tersebut. Ini mengikuti prinsip "DRY" (Don't Repeat Yourself - Jangan Mengulang Diri Sendiri).
  2. Membuat Kode Lebih Terstruktur: Fungsi membantu memecah program besar menjadi bagian-bagian yang lebih kecil dan mudah dikelola, seperti bab-bab dalam sebuah buku.
  3. Memudahkan Perawatan: Jika ada kesalahan atau perlu perubahan pada suatu tugas, kita hanya perlu mengubah kode di dalam satu fungsi, bukan mencari dan mengubahnya di banyak tempat.
  4. Meningkatkan Keterbacaan Kode: Dengan nama fungsi yang jelas, kita bisa memahami apa yang dilakukan sebuah program hanya dengan melihat nama-nama fungsinya, tanpa harus membaca detail setiap baris kode.
  5. Kode Bisa Digunakan Kembali (Reusability): Fungsi yang sudah dibuat bisa digunakan kembali di proyek program lain, menghemat waktu dan tenaga.

Membuat dan Memanggil Fungsi Sederhana: "Sapaan Pagi"

Untuk membuat sebuah fungsi di Python, kita menggunakan kata kunci def (kependekan dari "define" atau "mendefinisikan"), diikuti dengan nama fungsi, lalu tanda kurung (), dan diakhiri dengan tanda titik dua (:). Semua baris kode yang menjadi bagian dari fungsi harus menjorok ke dalam (indentasi). 

# Ini adalah definisi fungsi sederhana
def sapa_pagi():
    # Baris ini menjorok, berarti bagian dari fungsi sapa_pagi()
    print("Selamat pagi, dunia Python!")
    print("Semoga harimu menyenangkan.")

# Memanggil fungsi (menggunakan atau menjalankan fungsi)
print("--- Mulai Program ---")
sapa_pagi() # Ini akan menjalankan kode di dalam fungsi sapa_pagi()
print("--- Selesai Sapaan ---")
sapa_pagi() # Kita bisa memanggilnya lagi!
print("--- Program Berakhir ---")

Outputnya:

--- Mulai Program ---
Selamat pagi, dunia Python!
Semoga harimu menyenangkan.
--- Selesai Sapaan ---
Selamat pagi, dunia Python!
Semoga harimu menyenangkan.
--- Program Berakhir ---

Penjelasan Kode:

Pada baris def sapa_pagi():, kita memberi tahu Python bahwa kita sedang mendefinisikan sebuah fungsi bernama sapa_pagi. Dua baris print() di bawahnya yang menjorok ke dalam (indentasi) adalah "isi" dari fungsi tersebut.

Ketika kita menulis sapa_pagi(), ini berarti kita "memanggil" atau "menjalankan" fungsi sapa_pagi. Python akan langsung melompat ke definisi fungsi tersebut, menjalankan semua kode yang menjorok di dalamnya, dan setelah itu kembali melanjutkan kode di bagian program yang memanggilnya. Anda bisa melihat kita memanggil fungsinya dua kali, dan setiap kali dipanggil (ditulis), kedua pesan sapaan dicetak.

Fungsi dengan Parameter (Argumen): "Sapaan Personal"

Seringkali, fungsi perlu menerima informasi dari luar agar bisa melakukan tugasnya. Informasi ini disebut parameter (atau argumen ketika kita memberikannya saat memanggil fungsi). Parameter ditulis di dalam tanda kurung () setelah nama fungsi. Mereka bertindak seperti variabel di dalam fungsi yang menyimpan informasi yang dilewatkan. 

# Fungsi yang menerima satu parameter: 'nama'
def sapa_personal(nama):
    print(f"Halo, {nama}!")
    print("Apa kabar hari ini?")

# Fungsi yang menerima dua parameter: 'pesan' dan 'pengirim'
def kirim_pesan_dari(pesan, pengirim):
    print(f"Pesan dari {pengirim}: {pesan}")

# Memanggil fungsi dengan memberikan argumen
print("--- Sapaan untuk Budi ---")
sapa_personal("Budi") # Memberikan string "Budi" sebagai argumen untuk parameter 'nama'

print("\n--- Sapaan untuk Siti ---")
sapa_personal("Siti") # Memberikan string "Siti"

print("\n--- Pesan dari Aplikasi ---")
kirim_pesan_dari("Jadwal ujian diubah.", "Guru BK") # Memberikan dua argumen

Outputnya:

--- Sapaan untuk Budi ---
Halo, Budi!
Apa kabar hari ini?

--- Sapaan untuk Siti ---
Halo, Siti!
Apa kabar hari ini?

--- Pesan dari Aplikasi ---
Pesan dari Guru BK: Jadwal ujian diubah.

Penjelasan Kode:

Pada def sapa_personal(nama):, nama adalah parameter. Ketika sapa_personal("Budi") dipanggil, string "Budi" akan disimpan ke dalam parameter nama di dalam fungsi. Jadi, di dalam fungsi, parameter nama berisi "Budi".

Untuk kirim_pesan_dari(pesan, pengirim), ada dua parameter. Urutan argumen saat memanggil fungsi sangat penting: argumen pertama ("Jadwal ujian diubah.") akan masuk ke parameter pertama (pesan), dan argumen kedua ("Guru BK") akan masuk ke parameter kedua (pengirim).

Fungsi yang Mengembalikan Nilai (return): "Menghitung dan Memberi Hasil"

Sebuah fungsi tidak hanya bisa melakukan tugas, tetapi juga bisa "memberikan hasil" kembali ke bagian program yang memanggilnya. Ini dilakukan dengan kata kunci return. Ketika Python bertemu return di dalam fungsi, ia akan segera keluar dari fungsi tersebut dan "mengirimkan" nilai yang ditentukan kembali ke tempat fungsi itu dipanggil.

Nilai yang dikembalikan oleh fungsi biasanya disimpan ke dalam variabel. 

# Fungsi untuk menjumlahkan dua angka dan mengembalikan hasilnya
def jumlahkan(angka1, angka2):
    hasil_penjumlahan = angka1 + angka2
    return hasil_penjumlahan # Mengembalikan nilai hasil_penjumlahan

# Fungsi untuk menentukan status kelulusan dan mengembalikan teks
def cek_kelulusan(nilai, kkm):
    if nilai >= kkm:
        return "Lulus"
    else:
        return "Tidak Lulus"

# Memanggil fungsi dan menyimpan hasil kembaliannya
nilai_a = 10
nilai_b = 25
total = jumlahkan(nilai_a, nilai_b) # total akan berisi nilai yang dikembalikan oleh fungsi
print(f"Jumlah {nilai_a} dan {nilai_b} adalah: {total}")

nilai_ujian_siswa = 78
kkm_mata_pelajaran = 75
status_siswa = cek_kelulusan(nilai_ujian_siswa, kkm_mata_pelajaran)
print(f"Siswa dengan nilai {nilai_ujian_siswa} dinyatakan: {status_siswa}")

# Apa yang terjadi jika tidak ada return?
def fungsi_tanpa_return():
    print("Saya hanya mencetak sesuatu.")
    # Tidak ada 'return' di sini

hasil_tanpa_return = fungsi_tanpa_return()
print("Hasil dari fungsi tanpa return:", hasil_tanpa_return) # Output: None

Outputnya:

Jumlah 10 dan 25 adalah: 35
Siswa dengan nilai 78 dinyatakan: Lulus
Saya hanya mencetak sesuatu.
Hasil dari fungsi tanpa return: None

Penjelasan Kode:

Pada jumlahkan(angka1, angka2), fungsi ini menghitung angka1 + angka2 dan menyimpan hasilnya di hasil_penjumlahan. Kemudian, return hasil_penjumlahan mengirimkan nilai itu keluar dari fungsi. Ketika kita menulis total = jumlahkan(nilai_a, nilai_b), nilai 35 yang dikembalikan oleh fungsi akan disimpan ke dalam variabel total.

Pada cek_kelulusan, fungsi ini mengembalikan string "Lulus" atau "Tidak Lulus" tergantung pada kondisi.

Yang sering tidak disadari: jika sebuah fungsi tidak memiliki pernyataan return eksplisit (seperti fungsi_tanpa_return()), Python secara otomatis akan mengembalikan nilai khusus None. None berarti "tidak ada nilai" atau "kosong". Jadi, hasil_tanpa_return akan berisi None.

Parameter Default: Memberi Nilai Bawaan

Anda bisa memberikan nilai bawaan (default) untuk parameter. Jika saat memanggil fungsi, Anda tidak memberikan argumen untuk parameter tersebut, maka nilai defaultnya akan digunakan. Jika Anda memberikan argumen, nilai default akan diganti.

Parameter dengan nilai default harus diletakkan setelah parameter tanpa nilai default. 

# Fungsi dengan parameter default
def cetak_laporan(judul, halaman=1, penulis="Anonim"): # halaman dan penulis punya nilai default
    print(f"--- Laporan: {judul} ---")
    print(f"Penulis: {penulis}")
    print(f"Halaman: {halaman}")
    print("-" * 20)

# Memanggil fungsi dengan berbagai cara
print("Contoh 1: Semua argumen diberikan")
cetak_laporan("Studi Kasus AI", 15, "Dr. Informatika")

print("\nContoh 2: Menggunakan default halaman")
cetak_laporan("Pengantar Fisika", penulis="Prof. Einstein") # Halaman akan default 1

print("\nContoh 3: Menggunakan default penulis dan halaman")
cetak_laporan("Hasil Penelitian Kimia") # Halaman akan default 1, penulis akan default "Anonim"

Outputnya:

Contoh 1: Semua argumen diberikan
--- Laporan: Studi Kasus AI ---
Penulis: Dr. Informatika
Halaman: 15
--------------------

Contoh 2: Menggunakan default halaman
--- Laporan: Pengantar Fisika ---
Penulis: Prof. Einstein
Halaman: 1
--------------------

Contoh 3: Menggunakan default penulis dan halaman
--- Laporan: Hasil Penelitian Kimia ---
Penulis: Anonim
Halaman: 1
--------------------

Penjelasan Kode:

Pada def cetak_laporan(judul, halaman=1, penulis="Anonim"):, halaman memiliki nilai default 1 dan penulis memiliki nilai default "Anonim".

  • Pada panggilan pertama, semua argumen diberikan, sehingga nilai default diabaikan.
  • Pada panggilan kedua, halaman tidak diberikan, jadi Python menggunakan 1. penulis diberikan sebagai Prof. Einstein.
  • Pada panggilan ketiga, hanya judul yang diberikan, sehingga halaman dan penulis menggunakan nilai defaultnya.

Ini membuat fungsi lebih fleksibel, karena Anda tidak perlu selalu memberikan semua informasi jika sebagian sudah ada nilai umumnya.

Parameter Kata Kunci (Keyword Arguments): Memanggil dengan Nama

Saat memanggil fungsi, Anda bisa menyebutkan nama parameter yang ingin Anda berikan nilainya, bukan hanya urutannya. Ini disebut parameter kata kunci (keyword arguments). 

# Fungsi untuk mencetak biodata
def cetak_biodata(nama, usia, kota_asal):
    print(f"Nama: {nama}")
    print(f"Usia: {usia} tahun")
    print(f"Kota Asal: {kota_asal}")

# Memanggil fungsi dengan urutan argumen biasa (positional arguments)
print("--- Panggilan Positional ---")
cetak_biodata("Andi", 16, "Surabaya")

print("\n--- Panggilan Keyword ---")
# Memanggil fungsi dengan parameter kata kunci (urutan bisa acak!)
cetak_biodata(kota_asal="Bandung", usia=17, nama="Siti") # Urutan tidak masalah!

print("\n--- Kombinasi Positional dan Keyword ---")
# Positional arguments harus di depan keyword arguments
cetak_biodata("Rudi", kota_asal="Jakarta", usia=18)
# cetak_biodata(nama="Dewi", 15, "Medan") # Ini akan error! Positional setelah keyword

(Catatan: Baris cetak_biodata(nama="Dewi", 15, "Medan") sengaja dikomentari (#) karena akan menyebabkan error. Anda tidak bisa meletakkan argumen berdasarkan posisi setelah Anda mulai menggunakan argumen berdasarkan kata kunci.)

Outputnya:

--- Panggilan Positional ---
Nama: Andi
Usia: 16 tahun
Kota Asal: Surabaya

--- Panggilan Keyword ---
Nama: Siti
Usia: 17 tahun
Kota Asal: Bandung

--- Kombinasi Positional dan Keyword ---
Nama: Rudi
Usia: 18 tahun
Kota Asal: Jakarta

Penjelasan Kode:

Pada panggilan cetak_biodata(kota_asal="Bandung", usia=17, nama="Siti"), kita secara eksplisit menyebutkan parameter mana yang diisi. Keuntungannya:

  • Keterbacaan: Kode lebih mudah dibaca karena jelas parameter mana yang diberi nilai.
  • Fleksibilitas: Urutan argumen kata kunci tidak penting.

Namun, jika Anda mencampur argumen posisi (yang tidak menyebutkan nama parameter) dengan argumen kata kunci, argumen posisi harus selalu diletakkan di depan argumen kata kunci.

Mengembalikan Beberapa Nilai Sekaligus

Python memiliki kemampuan yang sangat keren untuk mengembalikan beberapa nilai dari sebuah fungsi sekaligus. Python akan secara otomatis "mengemas" nilai-nilai ini menjadi sebuah Tuple, dan Anda bisa "membongkar" Tuple tersebut menjadi beberapa variabel saat menerima hasilnya. 

# Fungsi untuk menghitung keliling dan luas persegi panjang
def hitung_persegi_panjang(panjang, lebar):
    keliling = 2 * (panjang + lebar)
    luas = panjang * lebar
    return keliling, luas # Mengembalikan dua nilai (dikemas dalam Tuple)

# Memanggil fungsi dan menerima beberapa nilai sekaligus
p = 10
l = 5
k_persegi, l_persegi = hitung_persegi_panjang(p, l) # Membongkar tuple yang dikembalikan
print(f"Persegi panjang dengan panjang {p} dan lebar {l}:")
print(f"Kelilingnya: {k_persegi}")
print(f"Luasnya: {l_persegi}")

print("\n")

# Contoh lain: Mendapatkan detail data siswa
def get_data_siswa(nama_lengkap):
    # Logika untuk mencari data siswa dari database (bayangkan saja)
    usia = 17
    alamat = "Jl. Merpati No. 5"
    return nama_lengkap, usia, alamat

siswa_nama, siswa_usia, siswa_alamat = get_data_siswa("Luna Maya")
print(f"Nama: {siswa_nama}, Usia: {siswa_usia}, Alamat: {siswa_alamat}")

Outputnya:

Persegi panjang dengan panjang 10 dan lebar 5:
Kelilingnya: 30
Luasnya: 50

Nama: Luna Maya, Usia: 17, Alamat: Jl. Merpati No. 5

Penjelasan Kode:

Pada return keliling, luas, Python secara internal mengemas keliling dan luas menjadi sebuah Tuple (keliling, luas). Ketika kita menulis k_persegi, l_persegi = hitung_persegi_panjang(p, l), Python secara otomatis "membongkar" Tuple yang dikembalikan: nilai pertama (keliling) masuk ke k_persegi, dan nilai kedua (luas) masuk ke l_persegi. Ini adalah cara yang sangat rapi untuk mengembalikan beberapa hasil dari sebuah fungsi.

Variabel Lokal vs. Global: Wilayah Kekuasaan Variabel

Kita sudah belajar tentang variabel. Sekarang penting untuk memahami di mana variabel-variabel itu "hidup" atau "berkuasa". Konsep ini disebut ruang lingkup (scope).

  • Variabel Lokal: Variabel yang dibuat di dalam sebuah fungsi hanya bisa diakses dan digunakan di dalam fungsi tersebut. Setelah fungsi selesai dijalankan, variabel lokal itu "mati" dan memorinya dibersihkan. Ibaratnya, itu adalah barang milik rumah (fungsi) itu sendiri.
  • Variabel Global: Variabel yang dibuat di luar fungsi (di tingkat utama program) bisa diakses dari mana saja, termasuk dari dalam fungsi. Ibaratnya, itu adalah barang umum di lingkungan tempat tinggal Anda. 
# Ini adalah variabel global
pesan_global = "Halo dari luar fungsi!"

def tampilkan_pesan():
    # Ini adalah variabel lokal
    pesan_lokal = "Halo dari dalam fungsi!"
    print(pesan_lokal)      # Bisa mengakses variabel lokal
    print(pesan_global)     # Bisa mengakses variabel global

def ubah_global_ini_tidak_disarankan():
    # Jika kita mencoba mengubah variabel global di dalam fungsi
    # Python akan membuat variabel lokal baru dengan nama yang sama
    # Kecuali jika kita menggunakan kata kunci 'global'
    pesan_global = "Pesan baru dari dalam fungsi (ini lokal!)"
    print(f"Di dalam fungsi, pesan_global (lokal) adalah: {pesan_global}")

def ubah_global_dengan_global_keyword():
    global pesan_global # Memberi tahu Python bahwa kita mau mengubah variabel global
    pesan_global = "Pesan baru diubah oleh fungsi dengan keyword global!"
    print(f"Di dalam fungsi, pesan_global (global) adalah: {pesan_global}")


# Panggil fungsi-fungsi
print("--- Scope Variabel ---")
print(f"Awal, pesan_global: {pesan_global}") # Akses global

tampilkan_pesan() # Fungsi mengakses lokal dan global

print(f"Setelah tampilkan_pesan, pesan_global: {pesan_global}") # Global masih sama

ubah_global_ini_tidak_disarankan() # Fungsi mencoba mengubah global (tapi membuat lokal baru)
print(f"Setelah ubah_global_ini_tidak_disarankan, pesan_global: {pesan_global}") # Global TIDAK berubah!

ubah_global_dengan_global_keyword() # Fungsi mengubah global dengan keyword 'global'
print(f"Setelah ubah_global_dengan_global_keyword, pesan_global: {pesan_global}") # Global BERUBAH!

Outputnya:

--- Scope Variabel ---
Awal, pesan_global: Halo dari luar fungsi!
Halo dari dalam fungsi!
Halo dari luar fungsi!
Setelah tampilkan_pesan, pesan_global: Halo dari luar fungsi!
Di dalam fungsi, pesan_global (lokal) adalah: Pesan baru dari dalam fungsi (ini lokal!)
Setelah ubah_global_ini_tidak_disarankan, pesan_global: Halo dari luar fungsi!
Di dalam fungsi, pesan_global (global) adalah: Pesan baru diubah oleh fungsi dengan keyword global!
Setelah ubah_global_dengan_global_keyword, pesan_global: Pesan baru diubah oleh fungsi dengan keyword global!

Penjelasan Kode:

  • pesan_global adalah variabel global. tampilkan_pesan() bisa membacanya.
  • pesan_lokal hanya ada di dalam tampilkan_pesan(). Anda tidak bisa mencetak pesan_lokal di luar fungsi itu.
  • Yang sering tidak disadari: di ubah_global_ini_tidak_disarankan(), saat kita menulis pesan_global = "Pesan baru...", Python tidak mengubah variabel global yang sudah ada. Sebaliknya, ia membuat variabel lokal baru bernama pesan_global khusus untuk fungsi itu! Jadi, variabel global di luar tetap tidak berubah.
  • Untuk benar-benar mengubah variabel global dari dalam fungsi, Anda harus menggunakan kata kunci global di depan nama variabel tersebut, seperti pada ubah_global_dengan_global_keyword(). Namun, menggunakan global keyword untuk mengubah variabel global dari dalam fungsi sangat tidak disarankan untuk pemula karena dapat membuat program sulit dipahami dan rawan kesalahan. Lebih baik kembalikan nilai yang diubah dari fungsi menggunakan return.

Docstrings: Mendokumentasikan Fungsi (Mirip Komentar, Tapi Lebih Canggih)

Sama seperti komentar, Docstrings digunakan untuk menjelaskan kode Anda. Docstrings adalah string multi-baris (diapit tiga kutip """ atau ''') yang diletakkan di baris pertama setelah def fungsi. Mereka digunakan untuk menjelaskan apa yang fungsi lakukan, parameter apa yang diterimanya, dan nilai apa yang dikembalikannya.

Bedanya dengan komentar (#): Docstrings bisa diakses oleh Python dan tools lain (misalnya, saat Anda mengetik help(nama_fungsi) di Python Console). 

def hitung_luas_segitiga(alas, tinggi):
    """
    Fungsi ini menghitung luas segitiga.

    Args:
        alas (float/int): Panjang alas segitiga.
        tinggi (float/int): Tinggi segitiga.

    Returns:
        float: Luas segitiga.
    """
    luas = 0.5 * alas * tinggi
    return luas

# Memanggil fungsi
luas_s = hitung_luas_segitiga(10, 5)
print(f"Luas segitiga: {luas_s}")

# Melihat docstring (di Python console, ketik: help(hitung_luas_segitiga))
print("\n--- Docstring Fungsi ---")
print(hitung_luas_segitiga.__doc__) # Cara mengakses docstring secara langsung

Outputnya:

Luas segitiga: 25.0

--- Docstring Fungsi ---

    Fungsi ini menghitung luas segitiga.

    Args:
        alas (float/int): Panjang alas segitiga.
        tinggi (float/int): Tinggi segitiga.

    Returns:
        float: Luas segitiga.

Penjelasan Kode:

Docstring di dalam hitung_luas_segitiga menjelaskan fungsi tersebut secara detail. Ini sangat membantu orang lain (dan diri Anda sendiri di masa depan) memahami cara kerja fungsi tanpa harus membaca seluruh kode di dalamnya. Ini adalah praktik yang sangat baik dan wajib untuk kode yang profesional.

Apa yang Mungkin Tidak Disadari: Fungsi adalah "Warga Negara Kelas Satu"

Ini adalah konsep yang lebih canggih, tapi Python sangat fleksibel. Di Python, fungsi dianggap sebagai "warga negara kelas satu" (first-class citizens). Artinya, Anda bisa memperlakukan fungsi sama seperti Anda memperlakukan angka atau teks:

  • Anda bisa menyimpan fungsi di dalam variabel.
  • Anda bisa melewatkan fungsi sebagai argumen ke fungsi lain.
  • Anda bisa mengembalikan fungsi dari fungsi lain.

Ini akan membuka pintu ke konsep-konsep yang lebih maju seperti "fungsi tingkat tinggi" (Higher-Order Functions) yang akan Anda pelajari nanti. Untuk saat ini, cukup ketahui bahwa ini adalah salah satu alasan mengapa Python sangat fleksibel dan kuat. 

# Modifikasi sapa_pagi untuk mengembalikan nilai
def sapa_pagi_return():
    return "Selamat pagi, dunia Python! (dari fungsi return)"

def sapa_sore():
    return "Selamat sore!"

# Menyimpan fungsi ke variabel lain
pesan_salam = sapa_sore
print("Memanggil fungsi yang disimpan di variabel:", pesan_salam())

# Mengembalikan fungsi dari fungsi lain (contoh sederhana)
def pilih_sapaan(waktu):
    if waktu == "pagi":
        return sapa_pagi_return # Mengembalikan fungsi sapa_pagi_return (tanpa tanda kurung!)
    else:
        return sapa_sore # Mengembalikan fungsi sapa_sore

panggil_sapaan = pilih_sapaan("pagi")
print("Memanggil fungsi yang dikembalikan:", panggil_sapaan())

Outputnya:

Memanggil fungsi yang disimpan di variabel: Selamat sore!
Memanggil fungsi yang dikembalikan: Selamat pagi, dunia Python! (dari fungsi return)

Penjelasan Kode (setelah koreksi sapa_pagi):

Pada pesan_salam = sapa_sore, kita tidak memanggil sapa_sore(), melainkan hanya menyimpan "alamat" atau referensi ke fungsi itu sendiri ke dalam variabel pesan_salam. Jadi, ketika kita memanggil pesan_salam(), yang sebenarnya dipanggil adalah fungsi sapa_sore.

Demikian pula, pilih_sapaan("pagi") mengembalikan fungsi sapa_pagi_return itu sendiri, bukan hasil eksekusinya. Variabel panggil_sapaan sekarang "berisi" fungsi sapa_pagi_return. Ketika kita memanggil panggil_sapaan(), barulah fungsi sapa_pagi_return dijalankan dan mengembalikan "Selamat pagi, dunia Python! (dari fungsi return)".
