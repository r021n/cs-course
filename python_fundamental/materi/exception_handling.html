Exception Handling – Membuat Program Lebih Mampu Mengantisipasi Kesalahan

Kita sudah belajar tentang berbagai jenis error di Python, seperti TypeError, ValueError, atau ZeroDivisionError. Ketika error semacam ini terjadi, biasanya program kita akan langsung berhenti dan menampilkan pesan kesalahan yang panjang (yang disebut "stack trace"). Ini mungkin baik untuk programmer yang sedang mencari masalah, tetapi bagi pengguna biasa, ini bisa sangat membingungkan dan membuat program terasa tidak profesional.

Bayangkan Anda sedang menggunakan aplikasi kalkulator dan tiba-tiba aplikasi itu macet total hanya karena Anda salah memasukkan angka atau mencoba membagi dengan nol. Tentu tidak nyaman, bukan?

Exception Handling adalah mekanisme di Python (dan bahasa pemrograman lainnya) yang memungkinkan kita untuk mengantisipasi dan mengelola error yang mungkin terjadi saat program berjalan. Ini seperti memiliki "jaring pengaman" atau "asuransi" untuk bagian kode yang berpotensi menyebabkan masalah. Dengan begitu, alih-alih program berhenti tiba-tiba, kita bisa memberikan pesan yang lebih ramah, mencoba memperbaiki masalah, atau melanjutkan eksekusi program dengan cara yang terkontrol.

Blok utama yang digunakan untuk penanganan pengecualian adalah try dan except.

1. try dan except: Jaring Pengaman Kode Anda

Ide dasarnya sederhana:

  • Kita meletakkan kode yang berpotensi menyebabkan error di dalam blok try.
  • Jika terjadi error di dalam blok try, Python tidak akan langsung menghentikan program. Sebaliknya, ia akan "melompat" ke blok except.
  • Di dalam blok except, kita menulis kode yang akan menangani error tersebut (misalnya, menampilkan pesan kesalahan yang ramah kepada pengguna). 
# Contoh 1: Menghitung pembagian dengan angka dari pengguna
print("--- Contoh Pembagian Angka ---")
try:
    # Kode yang berpotensi error (jika penyebut 0 atau input bukan angka)
    bilangan_str = input("Masukkan bilangan yang akan dibagi: ")
    pembagi_str = input("Masukkan pembagi: ")

    bilangan = float(bilangan_str) # Bisa ValueError jika bukan angka
    pembagi = float(pembagi_str)   # Bisa ValueError jika bukan angka

    hasil = bilangan / pembagi    # Bisa ZeroDivisionError jika pembagi 0
    print(f"Hasil pembagian: {hasil}")
except ZeroDivisionError:
    # Kode ini dijalankan jika terjadi ZeroDivisionError
    print("Maaf, tidak bisa membagi dengan nol!")
except ValueError:
    # Kode ini dijalankan jika terjadi ValueError
    print("Input yang Anda masukkan harus berupa angka.")
except:
    # Ini adalah 'catch-all' untuk error lain yang tidak spesifik
    print("Terjadi kesalahan yang tidak terduga.")

print("Program selesai berjalan (walaupun ada error di atas!).")

Outputnya (Contoh Interaksi 1: Pembagi nol):

--- Contoh Pembagian Angka ---
Masukkan bilangan yang akan dibagi: 10
Masukkan pembagi: 0
Maaf, tidak bisa membagi dengan nol!
Program selesai berjalan (walaupun ada error di atas!).

Outputnya (Contoh Interaksi 2: Input bukan angka): 

--- Contoh Pembagian Angka ---
Masukkan bilangan yang akan dibagi: sepuluh
Masukkan pembagi: 2
Input yang Anda masukkan harus berupa angka.
Program selesai berjalan (walaupun ada error di atas!).

Outputnya (Contoh Interaksi 3: Tidak ada error):

--- Contoh Pembagian Angka ---
Masukkan bilangan yang akan dibagi: 100
Masukkan pembagi: 5
Hasil pembagian: 20.0
Program selesai berjalan (walaupun ada error di atas!).

Penjelasan Kode:

  1. try:: Python akan mencoba menjalankan semua kode di bawahnya.
  2. Jika, misalnya, pengguna memasukkan 0 sebagai pembagi, maka bilangan / pembagi akan menyebabkan ZeroDivisionError. Python akan segera "melompat" ke blok except ZeroDivisionError:.
  3. Pesan "Maaf, tidak bisa membagi dengan nol!" akan dicetak. Program tidak berhenti total. Ia akan melanjutkan eksekusi ke baris setelah blok try-except, yaitu print("Program selesai berjalan...").
  4. Jika pengguna memasukkan "sepuluh", float(bilangan_str) akan menyebabkan ValueError. Python melompat ke except ValueError: dan mencetak pesan yang sesuai.
  5. Blok except: tanpa nama error (yang paling bawah) adalah "jaring pengaman" terakhir. Ia akan menangkap semua jenis error lain yang tidak ditangani oleh except sebelumnya. Sebaiknya gunakan ini sebagai pilihan terakhir dan usahakan selalu menangkap error yang lebih spesifik.
  6. Jika tidak ada error sama sekali (misal: 100 dibagi 5), maka seluruh blok try akan dieksekusi, dan semua blok except akan dilewati.

2. Mendapatkan Informasi Detail Error (as e)

Seringkali, kita ingin tahu lebih detail tentang error yang terjadi. Anda bisa menangkap objek error itu sendiri dan menyimpannya dalam sebuah variabel (biasanya dinamakan e atau err). Objek ini berisi pesan error yang sebenarnya dan detail errornya. 

# Contoh: Mendapatkan pesan error dari sistem
print("--- Mendapatkan Pesan Error ---")
try:
    # Coba mengakses kunci dictionary yang tidak ada
    data = {"nama": "Budi"}
    umur = data["usia"] # Ini akan memicu KeyError

    # Coba membagi dengan nol
    # hasil_bagi = 10 / 0

except KeyError as e:
    # 'e' akan menyimpan objek KeyError
    print(f"Ups! Terjadi kesalahan terkait kunci. Kunci yang tidak ditemukan: {e}")
except ZeroDivisionError as e:
    # 'e' akan menyimpan objek ZeroDivisionError
    print(f"Error matematika: {e}")
except Exception as e: # 'Exception' adalah kelas induk dari semua error
    # Ini akan menangkap semua jenis error lain
    print(f"Terjadi error tidak terduga: {e}")

print("Lanjutkan setelah penanganan error.")

Outputnya (jika KeyError terjadi):

--- Mendapatkan Pesan Error ---
Ups! Terjadi kesalahan terkait kunci. Kunci yang tidak ditemukan: 'usia'
Lanjutkan setelah penanganan error.

Penjelasan Kode:

Ketika KeyError terjadi, except KeyError as e: akan menangkapnya. Variabel e sekarang berisi objek KeyError. Ketika kita mencetak e, Python secara otomatis menampilkannya sebagai pesan KeyError itu sendiri, yaitu 'usia' (kunci yang tidak ditemukan).

except Exception as e: adalah cara umum untuk menangkap semua jenis error yang diwarisi dari kelas Exception. Ini seperti "jaring pengaman universal". Meskipun praktis, sebaiknya tetap gunakan except spesifik terlebih dahulu untuk error yang paling Anda antisipasi.

3. Blok else: Kode yang Jalan Jika Tidak Ada Error

Blok else yang diletakkan setelah try dan except akan dieksekusi hanya jika tidak ada error yang terjadi di dalam blok try. Ini berguna untuk menempatkan kode yang hanya boleh dijalankan setelah operasi utama berhasil. 

# Contoh: Proses login sederhana
print("--- Contoh Login Sederhana ---")
username = input("Masukkan username: ")
password = input("Masukkan password: ")

try:
    if username == "admin" and password == "rahasia":
        print("Login berhasil!")
        data_user = {"level": "administrator", "status": "aktif"}
        # Misal di sini ada proses lain yang penting setelah login
        print("Memuat data pengguna...")
    elif username == "user" and password == "guest":
        print("Login berhasil sebagai user!")
        data_user = {"level": "user", "status": "aktif"}
    else:
        # Jika username/password salah, kita bisa memicu error buatan
        raise ValueError("Username atau password salah!") # Kita "membuat" error sendiri

except ValueError as e:
    print(f"Kesalahan login: {e}")
except Exception as e:
    print(f"Terjadi kesalahan tak terduga: {e}")
else:
    # Kode di sini hanya akan dijalankan jika TIDAK ADA error di blok try
    print(f"Selamat datang, {username}! Level Anda: {data_user['level']}")
    print("Dashboard sedang dimuat...")

print("Proses autentikasi selesai.")

Outputnya (Contoh Interaksi 1: Login berhasil sebagai admin): 

--- Contoh Login Sederhana ---
Masukkan username: admin
Masukkan password: rahasia
Login berhasil!
Memuat data pengguna...
Selamat datang, admin! Level Anda: administrator
Dashboard sedang dimuat...
Proses autentikasi selesai.

Outputnya (Contoh Interaksi 2: Login gagal):

--- Contoh Login Sederhana ---
Masukkan username: admin
Masukkan password: salah
Kesalahan login: Username atau password salah!
Proses autentikasi selesai.

Penjelasan Kode:

  1. Jika username dan password benar ("admin"/"rahasia" atau "user"/"guest"), maka try akan berhasil diselesaikan tanpa error.
  2. Setelah itu, Python akan langsung melanjutkan ke blok else. Di sana, kita bisa mencetak pesan selamat datang dan memuat data tambahan yang hanya relevan jika login sukses.
  3. Jika username atau password salah, kita sengaja memicu ValueError dengan raise ValueError(...). Ini akan membuat Python melompat ke blok except ValueError as e:, dan blok else akan dilewati.

Menggunakan else ini membantu memisahkan kode yang "mencoba" dan kode yang "berhasil tanpa masalah", membuat struktur program lebih jelas.

4. Blok finally: Kode yang Selalu Dijalankan

Blok finally adalah bagian kode yang akan dieksekusi SETIAP SAAT, tidak peduli apakah error terjadi di blok try atau tidak, dan tidak peduli apakah error itu ditangani oleh except atau tidak. Ini sangat berguna untuk membersihkan sumber daya, seperti menutup file atau koneksi database, yang harus selalu dilakukan terlepas dari apa pun yang terjadi selama eksekusi program. 

# Contoh: Simulasi pemrosesan data (dengan 'finally' untuk cleanup)
print("--- Contoh Proses Data dengan Cleanup ---")
try:
    nilai1_str = input("Masukkan angka pertama: ")
    nilai2_str = input("Masukkan angka kedua: ")

    nilai1 = float(nilai1_str)
    nilai2 = float(nilai2_str)

    hasil = nilai1 / nilai2
    print(f"Perhitungan berhasil: {hasil}")

except ValueError:
    print("Error: Pastikan input adalah angka yang valid.")
except ZeroDivisionError:
    print("Error: Tidak bisa membagi dengan nol.")
finally:
    # Kode di sini akan SELALU dieksekusi,
    # baik jika try berhasil, gagal, atau error tidak ditangani
    print("--- Proses pembersihan (cleanup) selesai. ---")
    print("Aplikasi siap untuk operasi berikutnya atau ditutup.")

print("Selesai blok try-except-finally.")

Outputnya (Contoh Interaksi 1: Sukses):

--- Contoh Proses Data dengan Cleanup ---
Masukkan angka pertama: 20
Masukkan angka kedua: 4
Perhitungan berhasil: 5.0
--- Proses pembersihan (cleanup) selesai. ---
Aplikasi siap untuk operasi berikutnya atau ditutup.
Selesai blok try-except-finally.

Outputnya (Contoh Interaksi 2: Error ditangani):

--- Contoh Proses Data dengan Cleanup ---
Masukkan angka pertama: lima
Masukkan angka kedua: 2
Error: Pastikan input adalah angka yang valid.
--- Proses pembersihan (cleanup) selesai. ---
Aplikasi siap untuk operasi berikutnya atau ditutup.
Selesai blok try-except-finally.

Outputnya (Contoh Interaksi 3: Error tidak ditangani (misal: TypeError lainnya)): 

--- Contoh Proses Data dengan Cleanup ---
Masukkan angka pertama: 10
Masukkan angka kedua: []
Error: Pastikan input adalah angka yang valid.
--- Proses pembersihan (cleanup) selesai. ---
Aplikasi siap untuk operasi berikutnya atau ditutup.
Selesai blok try-except-finally.

Penjelasan Kode:

  • Jika perhitungan berhasil, print("Perhitungan berhasil...") akan dijalankan, lalu blok finally akan dijalankan.
  • Jika terjadi ValueError atau ZeroDivisionError, blok except yang sesuai akan dijalankan, lalu blok finally akan dijalankan.
  • Bahkan jika terjadi error yang tidak ditangani oleh except manapun (seperti jika kita memasukkan [] sebagai nilai2_str yang akan menyebabkan TypeError), blok finally tetap akan dijalankan sebelum Python menghentikan program dan menampilkan stack trace untuk error yang tidak ditangani tersebut.

Inilah kekuatan finally: menjamin bahwa kode pembersihan selalu berjalan.

5. Membuat Error Anda Sendiri (raise)

Seperti yang kita lihat di contoh else tadi, kadang-kadang Anda sebagai programmer ingin secara sengaja "membuat" atau "memicu" sebuah error jika suatu kondisi tidak terpenuhi. Ini disebut melempar (raising) sebuah pengecualian. Anda menggunakan kata kunci raise diikuti dengan tipe error yang ingin Anda picu, dan pesan opsional.

Ini sangat berguna untuk validasi data atau ketika Anda menemukan situasi yang seharusnya tidak terjadi dalam program Anda. 

def hitung_nilai_akhir(tugas, uts, uas):
    if not (0 <= tugas <= 100 and 0 <= uts <= 100 and 0 <= uas <= 100):
        # Memicu ValueError jika ada nilai yang di luar rentang 0-100
        raise ValueError("Semua nilai (tugas, UTS, UAS) harus antara 0 dan 100.")
    
    if tugas < 0 or uts < 0 or uas < 0: # Ini juga akan ditangkap oleh ValueError di atas
        # Contoh lain, jika ingin lebih spesifik untuk negatif
        # raise ValueError("Nilai tidak boleh negatif.")
        pass # Lewati saja karena sudah ditangkap ValueError di atas

    rata_rata = (tugas * 0.3) + (uts * 0.3) + (uas * 0.4)
    return rata_rata

print("--- Menghitung Nilai Akhir ---")
try:
    nilai_tugas = int(input("Masukkan nilai tugas: "))
    nilai_uts = int(input("Masukkan nilai UTS: "))
    nilai_uas = int(input("Masukkan nilai UAS: "))

    nilai_akhir = hitung_nilai_akhir(nilai_tugas, nilai_uts, nilai_uas)
    print(f"Nilai akhir Anda: {nilai_akhir:.2f}")

except ValueError as e:
    # Menangkap ValueError yang mungkin berasal dari input() atau dari raise kita sendiri
    print(f"Input Error: {e}")
except Exception as e:
    print(f"Terjadi error tak terduga: {e}")

Outputnya (Contoh Interaksi 1: Input valid):

--- Menghitung Nilai Akhir ---
Masukkan nilai tugas: 80
Masukkan nilai UTS: 75
Masukkan nilai UAS: 90
Nilai akhir Anda: 82.50

Outputnya (Contoh Interaksi 2: Input tidak valid, memicu raise ValueError): 

--- Menghitung Nilai Akhir ---
Masukkan nilai tugas: 105
Masukkan nilai UTS: 70
Masukkan nilai UAS: 80
Input Error: Semua nilai (tugas, UTS, UAS) harus antara 0 dan 100.

Penjelasan Kode:

Dalam fungsi hitung_nilai_akhir, kita menggunakan if not (...) untuk memeriksa apakah semua nilai berada dalam rentang yang diharapkan (0-100). Jika ada nilai yang di luar rentang itu, kita secara manual memicu ValueError dengan raise ValueError(...). Pesan yang kita berikan akan menjadi pesan error tersebut.

Di luar fungsi, kita menggunakan try-except untuk menangkap ValueError ini (atau ValueError lainnya dari fungsi int(input())). Ini menunjukkan bagaimana Anda bisa mengontrol alur program dan memberikan umpan balik yang spesifik kepada pengguna saat ada masalah yang Anda deteksi.

Apa yang Perlu Diperhatikan: Jangan "Menelan" Error Begitu Saja!

Kesalahan umum bagi pemula (dan bahkan beberapa programmer berpengalaman) adalah menggunakan except yang terlalu umum atau bahkan except: kosong tanpa melakukan apa-apa. 

# SANGAT TIDAK DISARANKAN!
try:
    hasil = 10 / 0 # Ini akan menyebabkan ZeroDivisionError
except: # Ini akan menangkap SEMUA error tanpa mempedulikan tipenya
    pass # 'pass' berarti 'jangan lakukan apa-apa'

print("Program tetap berjalan, tapi kita tidak tahu apa yang salah!")

Penjelasan Kode:

Jika Anda menjalankan kode di atas, ZeroDivisionError akan terjadi, tetapi blok except kosong akan "menelan"nya. Program akan terus berjalan tanpa memberi tahu Anda bahwa ada masalah serius. Ini disebut "menelan error" (swallowing errors) dan sangat berbahaya karena menyembunyikan masalah yang mungkin menyebabkan perilaku aneh di kemudian hari.

Selalu usahakan untuk:

  1. Menangkap error yang spesifik (misal: except ValueError:).
  2. Menampilkan pesan yang informatif kepada pengguna (dan mungkin kepada programmer untuk debugging).
  3. Melakukan tindakan korektif jika memungkinkan.

Gunakan except Exception as e: sebagai "jaring pengaman terakhir", tetapi pastikan Anda setidaknya mencetak e agar tahu error apa yang terjadi.

Hirarki Pengecualian: Error itu "Bersaudara"

Di Python, semua error (pengecualian) diatur dalam sebuah struktur hirarki, seperti pohon keluarga. Semua pengecualian adalah "anak" dari kelas dasar BaseException, dan kebanyakan yang Anda temui adalah "anak" dari Exception.

  • Exception
    • ValueError
    • TypeError
    • ZeroDivisionError
    • KeyError
    • IndexError
    • Dll.

Ini berarti jika Anda menangkap except Exception:, Anda secara otomatis akan menangkap semua pengecualian "anaknya" juga, seperti ValueError, TypeError, dll. Itu sebabnya disarankan untuk menangkap yang paling spesifik terlebih dahulu, lalu yang lebih umum. 

try:
    # Beberapa kode yang bisa menyebabkan berbagai error
    # int("abc") # ValueError
    # 10 / 0 # ZeroDivisionError
    my_list = [1, 2]
    print(my_list[3]) # IndexError

except ValueError:
    print("Ditangkap: Kesalahan nilai!")
except IndexError:
    print("Ditangkap: Kesalahan indeks!")
except Exception as e: # Ini adalah catch-all untuk sisanya
    print(f"Ditangkap: Error tak terduga: {type(e).__name__}: {e}")

Penjelasan Kode:

Urutan except sangat penting. Python akan mencoba mencocokkan error yang terjadi dengan blok except secara berurutan. Begitu menemukan kecocokan pertama, ia akan menjalankan blok tersebut dan melewati blok except lainnya. Jadi, jika IndexError terjadi, except IndexError akan menangkapnya. Jika ZeroDivisionError terjadi dan tidak ada except ZeroDivisionError, maka except Exception akan menangkapnya.
